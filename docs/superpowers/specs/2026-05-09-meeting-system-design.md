# 企业视频会议系统 - 设计文档

## 1. 产品定位

私有化部署的企业视频会议系统，类腾讯会议功能，支持 500 人并发，Flutter 跨平台客户端。

## 2. 技术架构

### 2.1 整体架构

```
                    ┌─────────────┐
                    │  Nginx/LB   │
                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           │               │               │
    ┌──────▼──────┐ ┌──────▼──────┐ ┌──────▼──────┐
    │  信令服务器  │ │  信令服务器  │ │  信令服务器  │ (2-3台)
    └──────┬──────┘ └──────┬──────┘ └──────┬──────┘
           │               │               │
    ┌──────▼───────────────▼───────────────▼──────┐
    │              mediasoup SFU 集群             │ (3-5台)
    └──────────────────────────────────────────────┘
                            │
                    ┌───────▼───────┐
                    │  录制/存储节点  │
                    └───────────────┘
```

### 2.2 技术选型

| 模块 | 技术 | 说明 |
|:---|:---|:---|
| 跨平台客户端 | Flutter 3.x | iOS/Android/PC |
| 实时通信 | WebRTC | 浏览器原生支持 |
| SFU 服务器 | mediasoup | 开源高性能，支持 500+ 人 |
| 信令服务器 | Go / Node.js | WebSocket 处理信令 |
| 后端 API | Go / Node.js | 用户管理、会议管理 |
| 存储 | MinIO / 阿里OSS | 会议录制存储 |
| 负载均衡 | Nginx | TCP/UDP 负载均衡 |

### 2.3 数据流

1. **加入会议**：客户端 → 信令服务器(WS) → 返回 room 信息
2. **媒体协商**：客户端 ↔ mediasoup (DTLS/SRTP)
3. **视频路由**：发布者 → SFU → 订阅者
4. **录制**：SFU 混流 → 录制节点 → 存储

## 3. 功能模块

### 3.1 会议管理

| 功能 | 描述 |
|:---|:---|
| 创建会议 | 生成唯一会议号，设置密码（可选） |
| 加入会议 | 输入会议号 + 密码加入 |
| 预约会议 | 设定时间、主题、参会人 |
| 会议列表 | 正在进行的会议、历史会议 |

### 3.2 实时通信

| 功能 | 描述 |
|:---|:---|
| 视频通话 | 画廊视图，最多显示 25 个视频 tile |
| 语音通话 | 纯音频模式，节省带宽 |
| 屏幕共享 | 主持人共享桌面/窗口 |
| 静音控制 | 主持人可静音全员/单人 |
| 摄像头控制 | 开关自己的摄像头 |

### 3.3 互动功能

| 功能 | 描述 |
|:---|:---|
| 文字聊天 | 实时文字消息 |
| 举手 | 观众举手发言 |
| 问答 | 观众提问，主持人筛选回答 |

### 3.4 录制管理

| 功能 | 描述 |
|:---|:---|
| 录制控制 | 主持人开始/停止录制 |
| 录制存储 | 录制文件存储到 OSS |
| 回放 | 支持录制回放（MP4） |

### 3.5 用户管理

| 功能 | 描述 |
|:---|:---|
| 用户体系 | 企业用户体系，对接 LDAP/SSO |
| 角色控制 | 主持人、参会者、观察者 |
| 会议控制 | 踢人、设主持人、转移主持人 |

## 4. MVP 范围（10 人以内先跑通）

### 优先级 P0（必须）

1. 创建会议、加入会议
2. 视频通话（9 人以内画廊视图）
3. 语音通话
4. 屏幕共享
5. 文字聊天
6. 主持人控制（静音、踢人）
7. 录制功能

### 优先级 P1（扩展到 100 人）

8. 举手发言
9. 问答功能
10. 会议预约
11. 用户管理

### 优先级 P2（500 人）

12. SFU 集群扩展
13. 带宽适应（分级画质）
14. 性能优化

## 5. 接口设计

### 5.1 REST API

| 接口 | 方法 | 描述 |
|:---|:---|:---|
| `/api/meeting/create` | POST | 创建会议 |
| `/api/meeting/:id` | GET | 获取会议信息 |
| `/api/meeting/:id/join` | POST | 加入会议 |
| `/api/meeting/:id/leave` | POST | 离开会议 |
| `/api/recording/:id` | GET | 获取录制文件 |

### 5.2 WebSocket 信令

| 事件 | 方向 | 描述 |
|:---|:---|:---|
| `join` | C→S | 加入房间 |
| `leave` | C→S | 离开房间 |
| `offer/answer` | C↔S | WebRTC 媒体协商 |
| `ice-candidate` | C↔S | ICE 候选 |
| `mute/unmute` | C→S | 静音/取消静音 |
| `kick` | S→C | 被踢出 |
| `chat` | C→S→C | 聊天消息 |
| `recording` | S→C | 录制状态通知 |

## 6. 数据库设计

### 6.1 核心表

- `users` - 用户表
- `meetings` - 会议表
- `meeting_participants` - 参会人表
- `recordings` - 录制表

## 7. 部署要求

### 7.1 硬件（500 人）

| 组件 | 数量 | 配置 |
|:---|:---|:---|
| Nginx | 2 台 | 4C8G |
| 信令服务器 | 3 台 | 8C16G |
| mediasoup SFU | 5 台 | 32C64G, 100Mbps |
| 后端 API | 2 台 | 8C16G |
| 存储 | 1 台 | 4C8G, 2TB+ |

### 7.2 带宽

- 每人上行 1Mbps（视频）→ SFU 需要 500Mbps
- 每人下行 1Mbps → SFU 需要 500Mbps
